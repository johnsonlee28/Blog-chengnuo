TOOLS.md

给 OpenClaw 配了六个 Skill，第二天它全忘了——原来少了这一步

上周干了一件特别有成就感的事。 花了一整个晚上，给猫巴士（AI助手）装了六个新 Skill。搜索的、画图的、语音合成的、读网页的，一个个配好 API Key，逐个测试通过。 那种感觉，就像给新员工准备好了工位、电脑、门禁卡、内部通讯录，连咖啡杯都摆上了。 心想：这下齐活了，明天开始大干一场。 第二天。 “帮忙搜一下 XXX。” 猫巴士回：“好的，让我用 Google 搜索……” 昨晚装的 Tavily 呢？那个专门给 AI 用的搜索引擎呢？ “帮我生成一张图片。” 又开始用最基础的方式瞎搞。 六个 Skill，一个都没用上。 新员工第一天上班，两手空空，跟你说：“工位？什么工位？电脑？没看到啊。” 当时真的有点崩溃。 这员工不是故意的，是真的失忆 冷静下来想了想，问题出在哪？ Skill 确实装好了，测试也通过了。问题不在工具本身，在于 AI 根本不知道自己有这些工具。 因为 AI 有一个跟人类完全不同的特点：它没有"昨天"。 每次新会话开始，AI 醒过来的时候，脑子是空的。不记得昨天装了什么，不记得上周配了什么，甚至不记得五分钟前另一个对话里发生的事。 想象一下：你的新员工每天早上走进办公室，都是全新的一天。不记得昨天同事教过他什么，不记得上周开会布置了什么任务，甚至不记得自己的工位在哪。 那他怎么能正常上班？ 靠一样东西：写在桌上的便签。 OpenClaw 每次启动，会自动加载几个核心文件： AGENTS.md — 相当于员工手册，告诉它该怎么干活 SOUL.md — 相当于岗位说明书，决定它的风格和态度 TOOLS.md — 相当于工位上的工具清单，记录有什么家伙事儿 MEMORY.md — 相当于工作笔记本，积累的经验教训 这些文件就是贴在工位上的便签。AI 每天"上班"第一件事，就是把这些便签全读一遍。 便签上写了的，它知道。没写的，就跟不存在一样。 所以问题不是"Skill 没装好"，而是装完之后没在便签上写一笔。 相当于给新员工配了一整套装备，但全锁在隔壁房间，门上也没贴条。他每天来上班，看到的是一张空桌子。 想通这一点，解决方案就清楚了。 最笨的办法，往往最有效 第一件事，打开 TOOLS.md，把每个 Skill 的信息写进去。 不用写得多精致，关键三样东西：干什么用、什么时候该用、配置在哪。 ## Tavily（AI搜索） - 路径：skills/tavily/SKILL.md - 触发词：搜索、查资料、找信息 - API Key：存于 op://Agent-All/tavily-op/password - 用途：比 Google 更适合 Agent 的搜索引擎，1000次/月免费 ## Nano Banana Pro（AI画图） - 路径：skills/nano-banana-pro/SKILL.md - 触发词：画图、生成图片、做个封面 - 用途：用 Gemini 生成或编辑图片 就这么简单。相当于在新员工桌上贴了一张纸条： ...
