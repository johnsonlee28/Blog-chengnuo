上周干了一件特别有成就感的事。

花了一整个晚上，给猫巴士（AI助手）装了六个新 Skill。搜索的、画图的、语音合成的、读网页的，一个个配好 API Key，逐个测试通过。

那种感觉，就像给新员工准备好了工位、电脑、门禁卡、内部通讯录，连咖啡杯都摆上了。

心想：这下齐活了，明天开始大干一场。（后来发现[派单命令也用错了](/posts/openclaw-multi-agent-dispatch/)，但那是另一个坑。）

第二天。

"帮忙搜一下 XXX。"

猫巴士回："好的，让我用 Google 搜索……"

昨晚装的 Tavily 呢？那个专门给 AI 用的搜索引擎呢？

"帮我生成一张图片。"

又开始用最基础的方式瞎搞。

六个 Skill，一个都没用上。

新员工第一天上班，两手空空，跟你说："工位？什么工位？电脑？没看到啊。"

当时真的有点崩溃。

## 这员工不是故意的，是真的失忆

冷静下来想了想，问题出在哪？

Skill 确实装好了，测试也通过了。问题不在工具本身，在于 AI 根本不知道自己有这些工具。

因为 AI 有一个跟人类完全不同的特点：**它没有"昨天"。**

每次新会话开始，AI 醒过来的时候，脑子是空的。不记得昨天装了什么，不记得上周配了什么，甚至不记得五分钟前另一个对话里发生的事。

想象一下：你的新员工每天早上走进办公室，都是全新的一天。不记得昨天同事教过他什么，不记得上周开会布置了什么任务，甚至不记得自己的工位在哪。

那他怎么能正常上班？

靠一样东西：**写在桌上的便签。**

OpenClaw 每次启动，会自动加载几个核心文件：

- **AGENTS.md** — 相当于员工手册，告诉它该怎么干活
- **SOUL.md** — 相当于岗位说明书，决定它的风格和态度
- **TOOLS.md** — 相当于工位上的工具清单，记录有什么家伙事儿
- **MEMORY.md** — 相当于工作笔记本（关于这套[三层记忆系统](/posts/openclaw-memory-system-three-layers/)的细节，单独写了一篇），积累的经验教训

这些文件就是贴在工位上的便签。**AI 每天"上班"第一件事，就是把这些便签全读一遍。**

便签上写了的，它知道。没写的，就跟不存在一样。

所以问题不是"Skill 没装好"，而是**装完之后没在便签上写一笔。**

相当于给新员工配了一整套装备，但全锁在隔壁房间，门上也没贴条。他每天来上班，看到的是一张空桌子。

想通这一点，解决方案就清楚了。

## 最笨的办法，往往最有效

第一件事，打开 `TOOLS.md`，把每个 Skill 的信息写进去。

不用写得多精致，关键三样东西：**干什么用、什么时候该用、配置在哪。**

```markdown
## Tavily（AI搜索）
- 路径：skills/tavily/SKILL.md
- 触发词：搜索、查资料、找信息
- API Key：存于 op://Agent-All/tavily-op/password
- 用途：比 Google 更适合 Agent 的搜索引擎，1000次/月免费

## Nano Banana Pro（AI画图）
- 路径：skills/nano-banana-pro/SKILL.md
- 触发词：画图、生成图片、做个封面
- 用途：用 Gemini 生成或编辑图片
```

就这么简单。相当于在新员工桌上贴了一张纸条：

> "抽屉里有计算器、订书机、扫描仪。计算器在左边第一格，订书机在右边，扫描仪插着 USB 线那个就是。"

五分钟搞定。下次 AI "上班"，读到 TOOLS.md，一眼知道自己有什么家伙事儿。

这是最笨的办法。但试过所有方法之后发现：**最笨的往往最稳。**

## 光知道有工具还不够，得知道什么时候用

TOOLS.md 解决了"有什么"，但还有一个问题："什么场景下该用什么"。

新员工知道抽屉里有计算器，但如果没人告诉他"做报销的时候用计算器"，他可能还是掰着手指头算。

这就需要在 `AGENTS.md` 里加一段调用规范：

```markdown
## Skill 使用规范
- 需要搜索信息 → 优先用 Tavily skill，找不到再用其他方式
- 需要生成图片 → 调用 nano-banana-pro skill
- 需要语音朗读 → 调用 sag skill（ElevenLabs TTS）
- 需要读网页内容 → 调用 agent-reach skill
```

相当于在员工手册里补了一页：遇到 A 类事情找张三，B 类事情找李四。不用每次都跑来问领导。

这一步的价值在于**优先级**。同一个需求可能有好几个 Skill 都能处理，写明先后顺序，AI 就不会每次随机抓一个。

## 再进一步：让它自己定期检查

前面两步都是"一次性配置"。配完之后，随着时间推移，会装新 Skill，会改旧配置，便签可能又跟不上了。

怎么办？让 AI 自己养成习惯，定期清点工具箱。

在 `HEARTBEAT.md` 里加一条定期任务：

```markdown
# Skill 健康检查（每周一）
- 列出所有已安装的 Skill
- 对比 TOOLS.md 的记录，有没有漏掉的
- 检查 API Key 是否还有效
- 发现新 Skill 没记录，自动补上
```

相当于给这个新员工立了一条规矩：**每周一早上，自己检查一遍工位，缺什么补什么，坏了什么报修什么。**

不用你催，它自己会做。

## 但说实话，不是每个人都需要做到这一步

如果只装了两三个 Skill，平时用得也不频繁，其实做完第一步（TOOLS.md 写死）就够了。花五分钟，问题解决 90%。

第二步适合 Skill 装得比较多，经常搞混优先级的情况。第三步适合已经把 OpenClaw 当主力工具在用，希望长期维护的人。

别为了"完整"把自己搞累了。工具是用来省事的，不是用来折腾的。选适合自己的深度就行。

## 一个值得养成的习惯

不管选哪个层级，核心习惯只有一个：

**装完即写档。**

每次装了一个新 Skill，花一分钟把它写进 TOOLS.md。就像新员工入职第一天，HR 带他认一遍办公室：这是你的工位，这是打印机，这是茶水间。走一遍，他就记住了。

不走这一遍，他能找到茶水间算运气好。

AI 的记忆力为零，但它的执行力无限。你只需要做好一件事：

把该记的东西，写在它每天醒来第一眼会看到的地方。

新员工不怕笨，怕的是桌上连张便签都没有。
